## ❌ DISADVANTAGES OF SALESFORCE DATA CLOUD

### 1. **EXTREMELY HIGH COST AT SCALE ($280-350M for 5 years)**

**Pricing Model:**
- Per-row pricing: $0.10-$0.50 per customer profile/month
- 50M profiles: $5-25M/month ($60-300M/year)
- CRM licenses on top: +$50-200/user/month
- Einstein Analytics: +$50K-$200K/month

**Toyota Scenario (50M profiles):**
- Year 1: $80M
- Years 2-5: $60-70M/year
- **5-Year Total: $280-350M**

**Custom Build Alternative:**
- Development: $3-5M
- Infrastructure: $600K-$1.8M/year
- Team: $1.5-2.5M/year
- **5-Year Total: $15-30M (80% savings)**

**❌ Toyota Disadvantage:** Massive financial burden for global company

### 2. **VENDOR LOCK-IN - DIFFICULT TO EXIT ($10-20M switch cost)**

**Why Lock-in Happens:**
- Data stored in proprietary Salesforce format
- Business processes depend on Salesforce workflows
- Integrations built to Salesforce APIs
- Team trained on Salesforce only
- Custom fields created in Salesforce schema

**Switching Cost:**
- Data extraction & transformation: $5-10M
- Building alternative platform: $3-5M
- Retraining staff: $500K-$1M
- Testing & validation: $1-2M
- **Total Exit Cost: $10-20M (often ≥ 2 years of licensing)**

**Custom Build Advantage:** Own your data, can migrate anytime

**❌ Toyota Disadvantage:** Trapped in long-term contract

### 3. **LIMITED CUSTOMIZATION FOR AUTOMOTIVE WORKFLOWS**

**Constraints:**
- Generic CDP, not automotive-specialized
- Vehicle ownership tracking requires custom fields
- Service appointment orchestration needs custom workflows
- Dealer POS integration requires custom code
- Trade-in valuation integration requires extensions
- Connected vehicle data integration requires custom work

**Example Limitations:**
- Can't easily model "household → vehicles → drivers"
- Service scheduling workflow is limited
- Finance/insurance integration not pre-built
- Dealer-specific logic not available
- Loyalty program customization limited

**Custom Build Advantage:** Build domain-specific models from day 1

**❌ Toyota Disadvantage:** Generic solution for unique automotive needs

### 4. **HIDDEN COSTS & SURPRISE BILLING (20-30% year-over-year increases)**

**Cost Escalation Sources:**
- Data volume growth (more profiles = higher cost)
- API call overages ($1-2 per 100 calls beyond limit)
- Storage expansion ($50K-$100K per 1TB)
- Einstein Analytics overages
- Premium support tier jump
- Professional services: $5K-$50K/day

**Real Toyota Scenario:**
- Year 1: Projected $50M, Actual $65M (+30%)
- Years 2-5: +$60-80M cumulative
- **Total Overrun: $70M+ over 5 years**

**❌ Toyota Disadvantage:** Budget uncertainty and finance pressure

### 5. **PERFORMANCE & LATENCY ISSUES AT SCALE**

**Real-World Issues:**
- Profile queries sometimes take 500ms-2s
- Complex segmentation queries timeout
- API rate limits force batching
- Query complexity limits (max 50K profiles per query)
- Concurrent user limits (expensive to increase)

**Toyota Use Case Problem:**
- Dealers need <500ms profile lookup
- Service advisors need <2s profile load
- Call center needs instant customer view
- Real-time recommendations must be <1s

**Custom Build Advantage:** PostgreSQL + Redis delivers <100ms guaranteed

**❌ Toyota Disadvantage:** Customer-facing app performance issues

### 6. **INFLEXIBLE FEATURE SET**

**Examples:**
- Identity resolution matching rules are limited
- Deduplication algorithm can't be customized
- Segmentation engine is fixed
- Customer profile schema has constraints
- Journey orchestration is restricted
- No access to core algorithms

**Toyota Limitations:**
- Can't implement vehicle-based deduplication
- Can't customize household linking logic
- Can't integrate proprietary ML models
- Can't modify consent enforcement

**Custom Build Advantage:** Own all algorithms

**❌ Toyota Disadvantage:** Forced to work within constraints

### 7. **DATA PRIVACY & RESIDENCY CONCERNS**

**Issues:**
- Data stored in Salesforce data centers
- Cannot store data on-premises
- Data residency options limited
- Salesforce can access data for support
- Subpoena/government requests possible
- No full encryption key control

**Toyota Concerns:**
- Customer data across 50+ countries
- Some regions require data residency
- China operations need data sovereignty

**Custom Build Advantage:** Full data sovereignty

**⚠️ Toyota Disadvantage:** Less data sovereignty

### 8. **SLOW TO ADAPT TO CHANGING REQUIREMENTS (8-27 months)**

**Timeline Reality:**
- Feature request to Salesforce: 0-6 months
- Roadmap inclusion: +3-6 months
- Development: +3-12 months
- General availability: +1-3 months
- **Total: 8-27 months for new capability**

**Custom Build Advantage:** Feature to deployment = 2-8 weeks

**⚠️ Toyota Disadvantage:** Business moves faster than platform

### 9. **RELEASE UPDATES CAN BREAK CUSTOMIZATIONS**

**Issues:**
- 3x yearly updates (Spring, Summer, Winter)
- APIs change, behavior changes
- Customizations may break silently
- Regression bugs not uncommon
- Testing required for each release

**Custom Build Advantage:** Control release cycle

**⚠️ Toyota Disadvantage:** Platform instability from updates

### 10. **LIMITED REAL-TIME CAPABILITIES**

**Salesforce Latency:**
- Event ingestion: 10-30 seconds delay
- Profile updates: 5-15 second propagation
- Real-time audience membership: 30-60 seconds
- Segment calculation: 5+ minutes

**Toyota Requirements: <1 second latency**

**Custom Build Advantage:** Kafka provides true real-time (<500ms)

**⚠️ Toyota Disadvantage:** Not suitable for real-time use cases

### 11. **POOR SUPPORT FOR ANALYTICS AT SCALE**

**Einstein Analytics Issues:**
- Expensive for large-scale analytics
- Limited query concurrency
- Data refresh delays (up to 24 hours)
- Limited historical data
- Querying large datasets is slow

**Alternative Cost:** Must add Snowflake/BigQuery = +$30K-$100K/month

**Custom Build Advantage:** Integrated data warehouse

**⚠️ Toyota Disadvantage:** Analytics becomes separate, expensive layer

### 12. **DIFFICULT TO INTEGRATE WITH LEGACY SYSTEMS**

**Toyota's Challenge:**
- LeadSquared ELMS (older system)
- CTDMS (custom Toyota system)
- TOPSERV (legacy service system)
- TFS (legacy finance system)
- TIBI (legacy insurance system)

**Integration Reality:**
- Salesforce connectors may not exist
- Custom integrations: $50K-$200K each
- Ongoing maintenance burden
- Error handling challenges

**Custom Build Advantage:** Build custom connectors easily

**⚠️ Toyota Disadvantage:** Legacy system integration expensive

### 13. **INSUFFICIENT FOR COMPETITIVE INTELLIGENCE**

**Salesforce Limitation:**
- Einstein AI is generic
- Can't train proprietary models
- Limited to pre-built algorithms
- Can't integrate external ML easily

**Toyota Advantage with Custom Build:**
- Train proprietary churn models
- Vehicle purchase probability models
- Connected vehicle data integration

**❌ Toyota Disadvantage:** Loses competitive advantage opportunity

### 14. **DEVELOPER EXPERIENCE IS POOR**

**Developer Friction:**
- Low-code/no-code is limiting for engineers
- Steep learning curve (Apex language)
- Debugging is difficult
- Testing framework is weak
- Not trendy with top talent

**Hiring Reality:**
- Hard to recruit top engineers
- High turnover (engineers want hands-on code)
- Can't attract from FAANG companies

**Custom Build Advantage:** Python/Java/Go = hot tech stack

**⚠️ Toyota Disadvantage:** Difficulty building world-class engineering team

✅ Quick time-to-market is critical (<12 months)
